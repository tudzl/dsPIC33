# dsPIC33AK I2S INMP441 Microphone Capture Project

## Overview

This project implements a real-time audio capture system using:
- **INMP441** MEMS microphone (I2S interface, 24-bit output)
- **dsPIC33AK512MPS506** Curiosity Nano board (MCU)
- **Python GUI** (PC-side) for waveform display, recording, and WAV export

The MCU captures audio samples via I2S (SPI host mode), packages them into UART frames, and streams to a PC over USB-UART at 500000 baud. DMA is used for UART transmission to minimize CPU load.

## Hardware Setup

```
INMP441 Mic          dsPIC33AK512MPS506 Curiosity Nano
-----------          -----------------------------------
  SCK  <-----------  RC1 (SPI1 SCK, 1.536MHz for 24KHz mode)
  SD   ----------->  RC4 (SPI1 SDI)
  WS   <-----------  RC6 (GPIO simulated LRCK)
  VDD  <-----------  3.3V
  GND  <-----------  GND
  L/R  <-----------  GND (Left channel)
```

Additional signals:
- **RD7 (INT2)**: External trigger to start/stop capture
- **RC3 (INT1/SW0)**: On-board button to toggle capture
- **RD0 (LED0)**: Toggles on each frame sent
- **RD2/RD3 (PWM1)**: 3.072MHz clock output
- **RC2 (PWM4)**: 2.8824MHz clock output

## Firmware (dsPIC33AK)

### File: `dsPIC33AK_Curiosity_Nano_i2s.X/main.c`

**Current Version: V1.08**

### I2S Audio Capture
- SPI1 in host mode with I2S audio extension enabled
- BCK (bit clock) = 1.536MHz (24KHz mode) or 3.072MHz (48KHz mode)
- WS (word select) generated by GPIO toggle in SCCP6 timer ISR
- 24-bit audio data read from SPI FIFO, sign-extended, shifted to 16-bit
- Mono capture: only left channel stored (WS toggle selects L/R)

### Double Buffering
- Two buffers: `mic_buffer_a[4000]` and `mic_buffer_b[4000]` (int16_t)
- ISR fills write buffer; when full, pointers swap (ping-pong)
- `mic_buffer_ready` flag signals main loop to transmit

### UART Transmission (DMA optimized)
- **Baud rate**: 500000 bps (8N1)
- **DMA zero-copy**: payload sent directly from `mic_read_ptr`
- Header (6B) and trailer (6B) sent blocking
- DMA Channel 0, trigger source 0x10, byte mode, source-increment

### Compile-time Options
| Macro | Default | Description |
|-------|---------|-------------|
| `MIC_BUFFER_SIZE` | 4000 | Samples per frame |
| `I2S_24Khz_mode_EN` | 1 | 24KHz mode (else 48KHz) |
| `Gain_shift_scale` | 7 | Right-shift for 24→16 bit conversion |
| `USE_DMA_UART_TX` | 1 | 1=DMA TX, 0=blocking TX |
| `AUDIO_UART_BAUD` | 500000 | UART baud rate |

### CPU Idle Statistics
- Main loop counts idle iterations vs total iterations
- INT2 trigger prints `[CPU]>>: idle=X/Y (Z%)` to UART

## Python GUI

### File: `mic_capture_gui.py`

**Current Version: V1.2**

### Features
- Serial port selection with auto-detection
- Configurable baud rate (500000 default)
- Configurable sample rate (24000 default, for WAV header)
- Configurable frame size (4000 default, must match FW)
- Real-time waveform display with Peak/RMS/dB info
- Start/Stop recording
- Save as WAV or raw PCM file

### Dependencies
```
pip install pyserial numpy
```
Tkinter is built-in with Python.

### Usage
```
python mic_capture_gui.py
```
1. Select COM port and set baud to 500000
2. Set FrameSize to match firmware `MIC_BUFFER_SIZE`
3. Click **Connect**
4. Trigger capture on MCU (INT2 or SW0)
5. Click **Start Recording** to capture
6. Click **Stop Recording** then **Save WAV**

## UART Frame Protocol

```
Byte offset  Field         Size    Description
-----------  -----         ----    -----------
0            Header        4B      AA 55 A5 5A (sync pattern)
4            Length        2B      Payload size in bytes (uint16 LE)
6            Payload       N*2 B   int16_t[N] PCM samples (LE)
6+N*2        Counter       4B      Frame count (uint32 LE)
10+N*2       Checksum      2B      Sum of payload + counter bytes (uint16 LE)
```

Default: N=4000, total frame = 8012 bytes

## Known Issues & Notes

1. **921600 baud not supported** by Curiosity Nano USB-UART bridge hardware
2. **500000 baud is minimum viable** for 24KHz/4000-sample configuration (96.1% utilization)
3. **DMA trigger source**: 0x10 (mapped to UART1 TX on this device, confirmed working)
4. **`0xFF80FFFFL` mask bug**: was corrupting audio data in V1.05, removed in V1.06+
5. **Frame drop fix (V1.08)**: `mic_buffer_ready` must not be cleared when DMA is busy
